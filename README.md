# ta-formalization-lab

## Purpose

This repository is a docs-first research and formalization lab for observing, extracting, and evaluating technical-analysis concepts.

It is not a trading system. It is not financial advice. It does not provide signals, execution logic, portfolio rules, or strategy guidance. It makes no performance or profitability claims.

A concept does not become project truth until it is written into the repository.

## Workflow

The working sequence is:

`observe` -> `extract` -> `decompose` -> `evaluate` -> `formalize/reject`

## Operating Frame

- `observe`: record what is explicitly present in a source.
- `extract`: isolate a candidate concept, rule, or claim.
- `decompose`: separate the concept into concrete components, conditions, and terms.
- `evaluate`: test whether the concept is precise, bounded, and distinguishable from interpretation.
- `formalize` or `reject`: either convert the concept into an explicit research definition or record why it does not qualify.
